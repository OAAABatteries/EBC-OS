  <action>
Audit the following components at 375px width and apply fixes ONLY where content demonstrably overflows or breaks (per D-13 -- no preemptive rules):

**1. `driver-kpi-grid`** -- Check if this class uses a fixed-column grid that overflows at 375px. If it uses `grid-template-columns` with fixed px values, add a `@media(max-width:399px)` override to reduce columns or switch to `1fr`. If it already uses `auto-fit` or `minmax()`, no change needed.

**2. ForemanView PortalTabBar** -- The `PortalTabBar` uses `.field-tab-bar` and `.field-tab-item`. Check if badge text or icon+label compress at 375px. If `.field-tab-item` label text truncates or overflows, add `overflow:hidden;text-overflow:ellipsis;white-space:nowrap` to `.field-tab-item` inside a `@media(max-width:399px)` block if not already present.

**3. EmployeeView form rows** -- Check `.emp-form-row` or similar class for flex/grid patterns. If label+input overflow at 375px, add `flex-wrap:wrap` inside a `@media(max-width:399px)` rule.

**4. Network banner** -- Verify `.network-banner` does not overflow at 375px. The `display:flex` with `gap:var(--space-2)` should wrap naturally. If the text is too long, add `flex-wrap:wrap` to the banner class (in the base rule, not a breakpoint -- wrapping is fine at all sizes).

**5. Verify at 768px and 1024px** -- These breakpoints already have rules. Check that no new Phase 6 additions (network banner) break at these sizes. The banner is a simple flex strip -- it should scale fine. No changes expected.

**Important constraints:**
- Do NOT add multi-column layouts for tablet (D-14 -- keep mobile stretch)
- Do NOT add preemptive breakpoints -- only fix actual overflow
- Landscape already has a media query -- verify no new issues
- If no overflow is found at any breakpoint, document it and make no changes to styles.js

Report each audit target and whether a fix was applied or not in the summary.
