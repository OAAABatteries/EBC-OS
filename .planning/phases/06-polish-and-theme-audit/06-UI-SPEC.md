## Spacing Scale

This phase uses the token scale already defined in `src/tokens.css`. No new spacing tokens are introduced.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline padding |
| `--space-2` | 8px | Network banner padding-y, compact element spacing |
| `--space-3` | 12px | Tight form row gaps |
| `--space-4` | 16px | Network banner padding-x, default element spacing |
| `--space-6` | 24px | Section padding |
| `--space-8` | 32px | Layout gaps |
| `--space-10` | 40px | Fixed-width numeric input (foreman hours row) |
| `--space-12` | 48px | Major section breaks |

Exceptions:
- Touch targets enforce `--touch-min: 44px` on all interactive elements (inherited from Phase 1 — not a new exception).
- Canvas elements (`field-signature-canvas`, `emp-jsa-canvas`) retain `touch-action: none` as an override to the new global `touch-action: manipulation` rule.
- `--space-3: 12px` — compact content padding token; inherited from Phase 1 token system; 3 × base-4 unit, grid-safe; used as card inner padding, form row gap, list item padding, and inline element gap across all three portals (DriverView, EmployeeView, ForemanView).
- `--space-10: 40px` — fixed-dimension token; inherited from Phase 1 token system; 10 × base-4 unit, grid-safe; used exclusively as the `width` of `.frm-hours-input` (the numeric time-entry input in foreman hours editing rows).

**Source:** `src/tokens.css` (direct code read); `src/styles.js` usage scan (direct code read).

## Color

This phase is multi-theme. The 60/30/10 split is declared per theme role, not a single hex value. The executor resolves colors at runtime through CSS custom properties.

### Theme Color Roles (apply across all 5 themes: Steel, Blueprint, Daylight, Matrix, Anime)

| Role | CSS Variable | Split | Usage |
|------|-------------|-------|-------|
| Dominant surface | `var(--bg)` | 60% | Page background, portal body |
| Secondary surface | `var(--bg2)` / `var(--bg3)` | 30% | Cards (`FieldCard`), sidebar, nav bar, header |
| Accent | `var(--amber)` | 10% | Primary CTA buttons, active tab indicator, focus ring, clock-in button, confirm actions |
| Destructive | `var(--red)` | — | Destructive actions only (delete, reject, danger buttons) |
| Success / reconnecting | `var(--green)` | — | Reconnect banner background tint (`var(--green-dim)`), success states |

Accent (`var(--amber)`) is reserved for: primary CTA buttons (`FieldButton` variant=primary), active tab highlight in `PortalTabBar`, the clock-in action button, confirm buttons in modal dialogs, and the `--focus-ring` token. Do not use `--amber` for decorative text, borders, or backgrounds outside these elements.

**`--accent` fix required (PLSH-01):** `var(--accent)` is referenced in `.driver-route-card`, `.driver-stop-badge`, `.emp-accent-label` (styles.js) but is NOT defined in any theme vars. Fix: add `"--accent": "var(--amber)"` to all 5 theme `vars` objects in `src/data/constants.js`. This single-file fix resolves all consumers simultaneously.

### Theme-Specific Dominant Colors (reference only — executor uses var() not hex)

| Theme | `--bg` | `--text` | Primary text contrast | Notes |
|-------|--------|----------|-----------------------|-------|
| Steel | `#06080c` | `#d4dae6` | ~10:1 | Indoor/vehicle |
| Blueprint | `#020a16` | `#bdddf0` | ~11:1 | Outdoor-optimized |
| Daylight | `#f2f3f7` | `#1a1d28` | ~13:1 | Outdoor-optimized |
| Matrix | `#000400` | `#00ff41` | ~12:1 | Indoor novelty |
| Anime | `#080414` | `#f0e4ff` | ~13:1 | Indoor novelty |

**Outdoor readability (D-03):** Daylight and Blueprint are the outdoor targets. Both primary text ratios already exceed 10:1. Audit focus must be on `--text2` values in these themes — minimum practical ratio for `--text2` is 3:1 against the theme `--bg` for field crew readability. Do not audit `--text3` for critical UI labels — it is intentionally low-contrast for muted/decorative elements.

**Source:** CONTEXT.md decisions D-03, D-07; RESEARCH.md contrast analysis; `src/data/constants.js` lines 17–185 (direct code read).

## Network Banner Interaction Contract

This is the primary new visual element in Phase 6.

### States

| State | Trigger | Display |
|-------|---------|---------|
| Hidden | `online === true` AND `wasOffline === false` | Not rendered — no DOM node |
| Offline | `online === false` | Full-width sticky strip below `PortalHeader` |
| Reconnecting | `online === true` AND `wasOffline === true` | Full-width strip; auto-hides after 5s (wasOffline auto-clears in hook) |

### Offline Strip Specification

- **Position:** `position: sticky` below the header strip (below the 54px header, above scroll body)
- **Height:** auto — padded by `--space-2` top/bottom, `--space-4` left/right
- **Background:** `var(--text3)` — uses the theme's muted background tone so it reads as a warning band without being alarm-red
- **Text color:** `var(--bg2)` — reversed contrast against the `--text3` background
- **Font:** `--text-base` (13px) at `--weight-bold` (700)
- **Icon:** WiFi-off icon from lucide-react, 16px, inline before text
- **Copy (offline):** `t("Offline") + " — " + pendingCount + " " + t("actions pending")`
- **Copy (reconnecting):** `t("Back online") + " — " + t("syncing") + " " + pendingCount + " " + t("actions")`

### CSS Class Names

```
.network-banner               — shared base
.network-banner--offline      — offline state styling
.network-banner--reconnecting — reconnect state styling
```

Classes go in `src/styles.js` field components section. Use `var()` tokens only — no hex literals.

**Source:** CONTEXT.md decisions D-09, D-10, D-11, D-12; RESEARCH.md architecture patterns (PortalHeader integration); RESEARCH.md code example for NetworkBanner CSS.

## Theme Switching Transition

- **Scope:** Global `*, *::before, *::after` rule in `src/index.css`
- **Properties animated:** `background-color`, `border-color`, `color`, `box-shadow`
- **Duration:** `var(--duration-micro)` (150ms)
- **Easing:** `var(--ease-standard)` (cubic-bezier(.4, 0, .2, 1))
- **Properties NOT animated:** `transform`, `opacity`, `width`, `height` — excluding these prevents animation interference with loading skeletons, shimmer, and hover transforms
- **Combined with touch-action rule:** Merge into a single `*, *::before, *::after { }` block to avoid duplicate selectors

**Source:** CONTEXT.md decisions D-05, D-08; RESEARCH.md architecture pattern "Theme Transition: Global CSS Rule"; RESEARCH.md pitfall 1.

## Touch and Scroll Contract

### Global Touch-Action Rule

- **Location:** `src/index.css` — single rule after body block
- **Selector:** `*, *::before, *::after`
- **Value:** `touch-action: manipulation`
- **Effect:** Eliminates 300ms tap delay on all elements globally (PLSH-02)
- **Overrides required (must verify):** `.field-signature-canvas` and `.emp-jsa-canvas` already have `touch-action: none` in `src/styles.js` (lines 317, 789). These override the global rule by cascade order — confirm they appear after the global import.

### Overscroll Behavior

- **Decision (D-17):** Keep `overscroll-behavior: none` on `body` in `src/index.css` — already present
- **Rationale:** Body uses `position: fixed; overflow: hidden`. Pull-to-refresh is not triggerable. PLSH-03 success criterion ("does not trigger browser pull-to-refresh") is satisfied by current state. No change needed.

**Source:** CONTEXT.md decisions D-16, D-17; RESEARCH.md recommendations; `src/index.css` (confirmed `overscroll-behavior: none` already global).

## Hover State Contract

- **Pattern:** Replicate existing `@media(hover:hover)` pattern from `.field-card` to additional interactive elements
- **Elements to update:** `.field-button` family (subtle background shift), `.field-tab-item` (background highlight), portal-specific buttons with unconditional hover states
- **Touch devices:** No hover effects — iOS/Android provide their own tap feedback. The `-webkit-tap-highlight-color: transparent` is already global.
- **Rule:** Any hover effect that changes background, border, or transform MUST be wrapped in `@media(hover:hover)` to prevent stuck-hover states after tap on touch devices.

**Source:** CONTEXT.md decision D-06; RESEARCH.md "Hover State Approach" recommendation; `src/styles.js` `.field-card` established pattern.

## Responsive Contract

### Viewport Breakpoints

| Breakpoint | Width | Status |
|------------|-------|--------|
| Small phone | max-width: 399px | Already in styles.js — covers 375px |
| Phone | max-width: 767px | Already in styles.js |
| Landscape | max-width: 900px + max-height: 500px + orientation: landscape | Already in styles.js |
| Tablet | max-width: 1024px | Already in App.css and styles.js |

### 375px Audit Targets

Add breakpoints only where content demonstrably breaks — no preemptive rules. Focus areas:

| Component | Risk | Resolution |
|-----------|------|------------|
| `driver-kpi-grid` | Grid tiles may compress | Verify at 375px; add `@media(max-width:399px)` column adjustment if needed |
| ForemanView tab bar | Badge count labels may compress | Check `PortalTabBar` at 375px; existing `@media(max-width:399px)` may suffice |
| EmployeeView form rows | Label + input may wrap awkwardly | Spot-check; fix only where overflow occurs |
| Modal widths | `min(580px, 94vw)` = 352.5px at 375px | Adequate — no fix needed |

### Tablet (768px–1024px)

Keep mobile stretch layout. Do not add multi-column for portals. Portals are phone-primary. Multi-column at 768px is premature for a field app. Existing `@media(max-width:1024px)` rules are sufficient.

### Orientation

Portrait is primary. Landscape must not overflow or break. Existing landscape media query (`max-width:900px, max-height:500px, orientation:landscape`) covers landscape edge cases.

**Source:** CONTEXT.md decisions D-13, D-14, D-15; RESEARCH.md "Responsive 375px Audit Scope".
