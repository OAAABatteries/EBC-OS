## Spacing Scale

The project spacing scale uses `--space-N` tokens defined in `src/tokens.css`. All values are px, no rem (established in Phase 1, enforced across codebase).

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Icon gaps, dot indicators, tight inline padding |
| --space-2 | 8px | Badge internal padding, flex row gaps in stat tiles |
| --space-3 | 12px | Info Card and Alert Card padding |
| --space-4 | 16px | Hero Card padding, default section padding |
| --space-5 | 20px | Between-card vertical gaps |
| --space-6 | 24px | Section header to content gaps |
| --space-8 | 32px | Between major content sections |
| --space-10 | 40px | Page-level top padding |
| --space-12 | 48px | Reserved for large layout gaps |

Exceptions:
- Touch targets: 44px minimum via `--touch-min` (not on scale, pre-existing token)
- Eagle logo in PortalHeader: 24px height (not a spacing token — it is an image sizing value)
- Alert dot indicator: 6px circle (sub-scale, consistent with Alert Card spec)
- PortalHeader height: 48px (pre-existing, locked — do not change)

Non-canonical values note: --space-3 (12px), --space-5 (20px), and --space-10 (40px) are intentional deviations from the strict 8-point canonical set. These are established tokens from Phase 1 and are not new additions in Phase 7. They are retained as-is for backward compatibility.

Source: tokens.css (direct read), design spec section 2.3, CONTEXT.md D-02

## Typography

All sizes are px. No rem. Exactly 4 sizes and 2 weights are used in this phase.

| Role | Token | Size | Weight | Line Height | Letter Spacing | Usage |
|------|-------|------|--------|-------------|----------------|-------|
| Label | --text-sm | 11px | 700 (--weight-bold) | 1.2 (--leading-tight) | 0.08em (--tracking-wider) | Uppercase category labels, badge text, stat tile labels, stat tile values |
| Body | --text-base | 13px | 400 (--weight-normal) | 1.5 (--leading-normal) | 0 | Descriptions, project names body, form values |
| Heading | --text-lg | 18px | 700 (--weight-bold) | 1.2 (--leading-tight) | 0 | "Schedule", "ALERTS", section headings, stat tile values (large) |
| Display / Hero | --text-display | 28px | 700 (--weight-bold) | 1.2 (--leading-tight) | 0 | Clock time hero, KPI hero numbers |

Consolidation notes (checker-required fixes applied):
- Stat tile label: was 9px — merged into 11px (--text-sm). Size difference is not perceptible at mobile scale and alignment with --text-sm improves system consistency.
- Stat tile value: was 20px at weight 800 — merged into 18px (--text-lg). Values now use --text-lg at --weight-bold.
- Card title: 13px (--text-base) at --weight-bold (700). No additional size token — uses same size as body with heavier weight.
- Weights collapsed from 5 (400, 600, 700, 800, 900) to 2 (400, 700). All former 600/800/900 usages now use 700 (--weight-bold).

Source: tokens.css (direct read), design spec sections 2.2 and 3.2, CONTEXT.md

## Color

All values reference EBC Brand theme CSS vars. The contract uses token names, not hex, so it applies correctly across all 8 themes at runtime.

| Role | EBC Brand Value | Token | Usage |
|------|----------------|-------|-------|
| Dominant (60%) | #0f1a24 | --bg | App background, page surface |
| Secondary (30%) | #152332 / #1b2d3f | --bg2 / --bg3 | Card backgrounds, sidebar, nav, inputs |
| Accent (10%) | #ff7f21 | --amber / --accent | Reserved list below |
| Success semantic | #10b981 | --green | Approved status, clocked-in, on-site |
| Warning semantic | #eab308 | --yellow | Certificate expiring, schedule change warning |
| Destructive | #ef4444 | --red | Denied status, overdue, destructive actions |
| Info semantic | #3b82f6 | --blue | New assignment alerts, in-transit |
| Primary text | #e8ecf2 | --text | Headers, data values, card titles |
| Secondary text | #8fa4ba | --text2 | Descriptions, body, project names |
| Tertiary text | #4d6478 | --text3 | Labels, timestamps, inactive tabs |
| Elevated surface | #1b2d3f | --bg3 | Hover states, active row background |
| Raised surface | #1e2d3b | --bg4 | Active press states |
| Glass border | rgba(255,255,255,0.06) | --glass-border | Subtle card borders |

Accent (--amber / --accent) reserved for:
- Active tab icon and label in PortalTabBar
- Left border on active shift card (3px border-left)
- FieldButton primary variant background
- Alert dot indicator color (warning type alerts)
- Week strip "today" highlight background (0.15 opacity)
- "Request Off" link text color in Schedule tab header
- PortalHeader bottom border (rgba(--accent, 0.15) — amber tint)
- Trade tag pill badges on Active Project card (0.1 opacity bg, full opacity text)
- Weekly progress bar fill gradient
- PremiumCard Hero variant accent left border (3px)
- Focus rings via --focus-ring token (var(--amber))

Second semantic usage pattern: Alert Card background uses `rgba(accent-color, 0.06)` and border `rgba(accent-color, 0.12)`. The accent-color here varies by alert type: --amber for warning, --red for error, --green for success, --blue for info.

## Focal Point

Primary screen focal point: the Home tab hero area — specifically the clock status block (ON CLOCK / OFF CLOCK) at --text-display (28px, --weight-bold) centered in the Hero Card. This is the first element the employee reads on open. All other content (stat tiles, alerts, active project) is secondary and flows below the hero block.

Design hierarchy on Home tab:
1. Hero Card — clock status + time display (--text-display, --accent for active state)
2. Stat tile row — Hours / Tasks / Pending (--text-lg at --weight-bold values, --text-sm labels)
3. Alert feed — time-sensitive notifications
4. Active project card — current assignment summary

Source: design spec section 2.2, CONTEXT.md D-01

## Interaction Contract

| Interaction | Behavior | Timing |
|------------|----------|--------|
| Tab switch | Immediate render, no loading flash | instant |
| Stat tile tap | Navigate to source detail view | instant |
| Alert card tap | Navigate to source tab | instant |
| Shift "Pick Up" tap | Show pending state on button, submit to Supabase, update status inline | --duration-micro (150ms) for visual press |
| Time-off "Request Off" tap | Open bottom sheet modal | --duration-state (300ms) slide up |
| Bottom sheet submit | Show FieldButton loading spinner, dismiss on success | --duration-micro |
| Credential card tap | Navigate to credential detail | instant |
| Eagle logo load error | Hide logo (existing onError fallback pattern — do not show broken image) | instant |
| Theme-aware eagle logo | CSS filter on ebc-eagle-white.png base asset for non-EBC themes needing contrast adjustment | no transition |
| Week strip day tap | Scroll to day's shifts (smooth scroll) | native scroll |
| FieldCard / PremiumCard press | No transform on touch devices (hover:hover gate — established Phase 2) | n/a |
| EmptyState action button | Must always be present — action prop required, never undefined | n/a |
| Supabase query | Loading state via AsyncState wrapper, error state with message + retry | n/a |

Touch target minimum: 44px via --touch-min on all interactive elements. No exceptions in this phase.

Global transition scope (established Phase 6): background-color, border-color, color, box-shadow only. No transform or opacity in global transition — they animate per-component.

Source: design spec section 6.3, CONTEXT.md D-03, D-07, D-09, Phase 2 decisions (hover:hover), Phase 6 decisions (transition scope)

## Accessibility Notes

- All new interactive elements: 44px minimum touch target via --touch-min
- Focus rings on all interactive elements via --focus-ring token (established Phase 1)
- Color is never the only differentiator for status — StatusBadge uses text label + color
- Alert types differentiated by dot color + text label (not color alone)
- Eagle logo: `alt="EBC Eagle"` on all img elements; hidden via CSS if load fails (not broken image shown)
- Lucide icons: aria-hidden="true" where decorative; meaningful icons get aria-label
- Bottom sheet modals: focus trap within sheet, Escape key closes, background not scrollable while open
