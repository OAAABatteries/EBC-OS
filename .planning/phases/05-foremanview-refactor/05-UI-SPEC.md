## Spacing Scale

All spacing in this phase MUST consume the token scale defined in `src/tokens.css`. No raw pixel values in new CSS classes.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, inline badge padding |
| `--space-2` | 8px | Compact element spacing, KPI label margin |
| `--space-3` | 12px | Input padding, list item inner spacing |
| `--space-4` | 16px | Default element spacing, card padding |
| `--space-6` | 24px | Section padding, card gap |
| `--space-8` | 32px | Layout gaps between major sections |
| `--space-10` | 40px | Tab bar height contribution |
| `--space-12` | 48px | Major section breaks |

Exceptions:
- Touch targets: minimum 44px height via `var(--touch-min)` — applies to ALL buttons, tab bar items, and interactive list rows (FRMN-03)
- Bottom tab bar: fixed height 56px to accommodate icon + label + safe-area inset (matches Phase 4 pattern)
- KPI grid gap: `var(--space-2)` (10px used in existing `.foreman-kpi-grid` — keep as-is, it already exists)

Source: `src/tokens.css` (Phase 1 output).

## Typography

All type in new CSS classes MUST reference tokens from `src/tokens.css`. No ad-hoc font sizes in new foreman-specific classes.

| Role | Token | Value | Weight Token | Weight | Line Height | Usage |
|------|-------|-------|-------------|--------|-------------|-------|
| KPI value / Display | `var(--text-display)` | 28px | `var(--weight-bold)` | 700 | `var(--leading-tight)` 1.2 | Dashboard KPI dollar amounts, percentages, crew counts (FRMN-04) |
| Heading / Section | `var(--text-lg)` | 18px | `var(--weight-bold)` | 700 | `var(--leading-tight)` 1.2 | Section titles, modal titles, card titles |
| Body / Interactive | `var(--text-base)` | 13px | `var(--weight-normal)` | 400 | `var(--leading-normal)` 1.5 | Table cells, button labels, form inputs, list body copy |
| Badge / KPI label | `var(--text-sm)` | 11px | `var(--weight-normal)` | 400 | `var(--leading-normal)` 1.5 | Status badges, KPI sublabels, timestamps |

Decision D-05 locked: single token `var(--text-display)` for all KPI primary values (not `var(--text-2xl)` — the token is `--text-display: 28px` in tokens.css). KPI label uses `var(--text-sm)`.

Existing `.foreman-kpi-value` uses 22px — this class MUST be updated to `font-size: var(--text-display)` in Plan 1 (CSS). Existing `.foreman-kpi-label` uses 9px — update to `font-size: var(--text-sm)`.

Numeric values (hours, budget figures, quantities) use `font-family: var(--font-mono)` — matches existing foreman pattern and Phase 3/4 precedent.

## Color

This project uses a multi-theme CSS variable system. No hex literals permitted in JSX or new CSS class declarations. All color references MUST use `var(--token-name)`.

| Role | Token | Resolves To (default dark theme) | Usage |
|------|-------|----------------------------------|-------|
| Dominant surface (60%) | `var(--bg)` | #06080c | Page background, portal shell |
| Secondary surface (30%) | `var(--bg2)` / `var(--bg3)` | #0c0f16 / #12161f | Cards, tab bar background, sidebar |
| Accent (10%) | `var(--amber)` | #e09422 (theme-resolved) | KPI values, active tab indicator, FieldButton primary, active nav icon |
| Destructive | `var(--red)` | theme-resolved | Danger FieldButton variant, denial status badges only |
| Text primary | `var(--text)` | theme-resolved | All body copy, headings |
| Text secondary | `var(--text2)` | theme-resolved | Sublabels, muted copy, timestamps |
| Text tertiary | `var(--text3)` | theme-resolved | Placeholder copy, disabled states |
| Glass surface | `var(--glass-bg)` | rgba(12,15,22,0.72) | Modal overlays, sheet panels |
| Glass border | `var(--glass-border)` | rgba(255,255,255,0.06) | Card borders, dividers |

Accent (`var(--amber)`) is reserved for:
1. KPI primary values (`.foreman-kpi-value`)
2. Active tab icon + label in PortalTabBar
3. FieldButton primary variant background
4. Hours and monetary figures using `var(--font-mono)`
5. Focus ring via `var(--focus-ring)`

Phase status colors — these tokens are already defined in all 8 themes in `constants.js`. MUST replace all hex literals in ForemanView JSX (FRMN-06):

| Stage | Token |
|-------|-------|
| Active | `var(--phase-active)` |
| Estimating | `var(--phase-estimating)` |
| Pre-construction | `var(--phase-pre-construction)` |
| Completed | `var(--phase-completed)` |
| Warranty | `var(--phase-warranty)` |
| In Progress | `var(--phase-in-progress)` |

Dynamic runtime exceptions (same precedent as Phase 3/4): progress bar `width` percentages, risk score opacity levels, and dynamically computed color interpolations are acceptable as inline styles because they are runtime-computed values, not static theme values. Document each exception in a JSX comment: `{/* dynamic: runtime value */}`.

## Tab Navigation Contract

Source: Decision D-01 (CONTEXT.md) — Claude's discretion on which 4-5 tabs are primary.

### Primary Tab Bar (bottom nav via PortalTabBar)

| Position | Tab Key | Label | Icon (lucide) | Badge Condition |
|----------|---------|-------|---------------|-----------------|
| 1 | `dashboard` | Dashboard | `LayoutDashboard` | false |
| 2 | `materials` | Materials | `Package` | pending material requests exist |
| 3 | `jsa` | JSA | `Shield` | no active JSA on current project |
| 4 | `reports` | Reports | `FileText` | false |
| 5 | `settings` | Settings | `Settings` | false |

### More Overflow (sheet panel via PortalTabBar built-in)

Remaining 8 tabs route through the More overflow menu:

| Tab Key | Label |
|---------|-------|
| `clock` | Clock |
| `team` | Team |
| `hours` | Hours |
| `drawings` | Drawings |
| `lookahead` | Lookahead |
| `documents` | Documents |
| `site` | Site |
| `notes` | Notes |

Rationale for primary selection: Dashboard is foreman's command center (viewed every session). Materials and JSA are high-frequency daily operations with time-sensitivity. Reports are the primary output artifact foreman produces. Settings is a permanent nav convention. Clock, Team, Hours are secondary because they are viewed once per day rather than multiple times.

Language toggle included in PortalHeader via `langToggle` prop — same segmented EN/ES control as Phase 4 (`emp-lang-switch` pattern). Foreman may operate bilingual crews and needs language access without going into Settings. Follow exact Phase 4 implementation.

## Interaction Contract

### Touch Targets
Every tappable element meets `var(--touch-min)` = 44px minimum height (FRMN-03):
- All FieldButton instances: enforced by component
- All PortalTabBar items: enforced by component
- Team member rows: `min-height: var(--touch-min)` via `frm-team-row` class
- Material request action rows: `min-height: var(--touch-min)`
- Drawing list items: `min-height: var(--touch-min)`
- Calendar/lookahead day cells: minimum 44px

### Focus States
All interactive elements use `var(--focus-ring)` — amber glow, 2px offset. Enforced via `.focus-visible:focus-visible` utility class on FieldButton, FieldInput, FieldSelect, tab bar items.

### Loading States
Follow Phase 3/4 skeleton pattern:
- Tab switch: show AsyncState with Skeleton shimmer while data loads
- Form submit: FieldButton `loading={true}` disables button and shows Loader2 spinner
- KPI cards: Skeleton blocks at exact KPI card dimensions while project data loads
- Shimmer animation gates on `@media (prefers-reduced-motion: no-preference)` — no animation for users who opt out

### Transitions
- Tab switch: `var(--transition-micro)` (150ms) on content fade
- Card hover (non-touch): `var(--transition-micro)` on background color (gate with `@media (hover: hover)`)
- PortalTabBar active indicator: `var(--transition-state)` (300ms)
- Modal open/close: `var(--transition-state)` (300ms) slide-up from bottom

### Budget Progress Bar
`.foreman-budget-fill` width is a runtime-computed inline style (percentage). This is an approved dynamic exception — add JSX comment: `{/* dynamic: budget % computed at runtime */}`. Color of fill uses `var(--phase-in-progress)` or `var(--red)` based on threshold — no hex.
