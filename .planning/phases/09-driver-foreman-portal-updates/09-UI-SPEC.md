## Spacing Scale

Declared values (all from tokens.css — do not use raw px in JSX):

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Icon gaps, dot indicators, tight inline pairs |
| --space-2 | 8px | Badge gaps, stat tile inner gap, shift-card row gaps |
| --space-3 | 12px | PremiumCard Info padding, alert card padding |
| --space-4 | 16px | PremiumCard Hero padding, section horizontal padding |
| --space-5 | 20px | Between section blocks within a tab |
| --space-6 | 24px | Section header top margins |
| --space-8 | 32px | Between major tab content sections |
| --space-10 | 40px | Empty state vertical centering pad |
| --space-12 | 48px | Tab bar height clearance bottom padding |

Exceptions:
- Touch targets: 44px minimum height on all tappable elements (--touch-min token)
- Stat tile row: `flex gap: var(--space-2)` (8px), tiles flex:1 equal width
- Filter chips row: `flex gap: var(--space-2)` (8px) with horizontal scroll, no wrap

**Source:** tokens.css, design spec §2.3, COMP-01 through COMP-05

## Typography

Use tokens.css type scale. No custom font-size values in JSX. Four sizes in active use:

| Role | Token | Size | Weight | Line Height | Use in Phase 9 |
|------|-------|------|--------|-------------|----------------|
| Display | --text-display | 28px | 900 (inline bold) | --leading-tight (1.2) | Driver route count hero, Foreman KPI heroes (budget, hours, crew count) |
| Section heading | --text-lg | 18px | --weight-bold (700) | --leading-tight (1.2) | "TODAY'S DELIVERIES", "PENDING REQUESTS", "CREW CERTIFICATIONS" section labels |
| Body / card title | --text-base | 13px | --weight-bold (700) for titles, --weight-normal (400) for body | --leading-normal (1.5) | Route card project name, request card employee name, cert card title, all body copy |
| Label / badge | --text-sm | 11px | --weight-bold (700) | --leading-tight (1.2) | Uppercase category labels, status badges, stat tile labels, timestamps |

Two weights only: --weight-normal (400) and --weight-bold (700).

## Color

EBC Brand theme is the default. All values reference CSS variables — no hex literals in JSX.

| Role | Token | EBC Value | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | --bg | #0f1a24 | App background, tab content areas |
| Secondary (30%) | --bg2 / --bg3 | #152332 / #1b2d3f | Card surfaces (PremiumCard Info = --bg2), elevated inputs = --bg3 |
| Accent (10%) | --accent (= --amber) | #ff7f21 | See reserved list below |
| Success semantic | --green | #10b981 | Clocked-in status, DELIVERED badge, approved cert badge, Approve action |
| Warning semantic | --yellow / --amber | #eab308 / #ff7f21 | Cert expiring-soon badge, overtime label, alert dot for schedule change |
| Destructive | --red | #ef4444 | Denied badge, expired cert badge, DENY action, overdue delivery |
| Muted text | --text3 | #4d6478 | Timestamps, inactive tab labels, section micro-labels |

**Accent (--accent) reserved for:**
- Active tab icon + label in PortalTabBar
- Hero card left border (border-left: 3px solid var(--accent)) on active/in-transit route stop
- CTA FieldButton primary variant background
- Stat tile value color for Pending count (pending items need urgency signal)
- Active shift card left border in foreman request cards
- Badge dot on shift pickup / overtime label text
- PortalHeader bottom border accent tint (portal-header-accent-border class)
- Filter chip active state background (rgba(--accent, 0.15)) + text

Second semantic color (--red) reserved for: Deny buttons (danger variant FieldButton), expired certs, denied requests.

**Source:** constants.js EBC theme, styles.js premium-card-* classes, design spec §2.1, CONTEXT.md D-11

## Layout Contracts

### Driver Portal

**Tab Structure (PortalTabBar, maxPrimary=4):**

| Position | Tab ID | Icon (Lucide) | Badge |
|----------|--------|---------------|-------|
| Primary 1 | home | Home | none |
| Primary 2 | route | Navigation | optimizedStops.length > 0 |
| Primary 3 | completed | CheckCircle | todayDelivered.length > 0 |
| Primary 4 (overflow trigger) | — | MoreHorizontal | — |
| More overflow | settings | Settings | none |

Default landing tab: `home` (state initial value: `"home"`, not `"route"`)

**Driver Home Tab — Layout (top to bottom):**

1. Route count hero (PremiumCard hero variant)
   - Greeting: "Good [Morning/Afternoon], {firstName}" — 11px, --text3, uppercase, var(--tracking-wider)
   - Hero number: total active stops — 28px, weight 900, --text, line-height 1.2
   - Sub-label: "STOPS TODAY" — 11px, --text3, uppercase
   - Left border: 3px solid var(--accent) if stops > 0, else var(--text3)

2. Stat tiles row — 3 tiles, flex with gap var(--space-2), each flex:1
   - Pending: count of optimizedStops not yet delivered — value color: var(--accent)
   - Completed: count of todayDelivered — value color: var(--green)
   - Miles: totalDistance.toFixed(1) — value color: var(--text)
   - Each tile tappable: Pending → route tab, Completed → completed tab, Miles → no-op (display only)

3. Alerts feed (AlertCard stack)
   - Section label: "ALERTS" — 11px, --text3, uppercase, var(--tracking-wider)
   - Max 3 AlertCards, newest first
   - "View All" link below (if count > 3): --accent color, --text-base, right-aligned or left-aligned inline
   - Alert types: warning (schedule change = amber dot), info (new assignment = blue dot), success (delivery approved = green dot)
   - Each AlertCard tappable → navigates to route tab

**Driver Route Tab — PremiumCard upgrade:**
- Replace FieldCard with PremiumCard info variant on each route stop card
- Active/in-transit stop: PremiumCard hero variant with accent left border
- Status badge progression: ASSIGNED → IN TRANSIT → DELIVERED (StatusBadge)
- Distance badge: 11px, --text3, "X.X mi" inline with MapPin 12px icon
- Drag handle: 44px touch target, visible drag indicator (≡ icon, --text3)

| Primary 3 | hours | Clock | none |

Default landing tab: `dashboard` (replaces current `clock` default)

**Foreman Dashboard Tab — Layout (top to bottom):**

1. Clock status hero (PremiumCard hero variant)
   - Clock status: "ON CLOCK" (--green) or "OFF CLOCK" (--text3) — 11px uppercase label
   - Duration since punch-in or "—" — 28px, weight 900, --text
   - Sub-label: "TAP TO CLOCK IN/OUT" — 11px, --text3, uppercase — entire card tappable, navigates to clock tab in More
   - Left border: 3px solid var(--green) if clocked-in, var(--text3) if off

2. KPI tiles row — 3 tiles, flex with gap var(--space-2), each flex:1
   - Budget: % used — color var(--accent) if > 90%, var(--green) if <= 90%
   - Hours: total hours today — color var(--text)
   - Crew: clocked-in crew count — color var(--green)
   - Each tappable: Budget → hours tab, Hours → hours tab, Crew → team tab

3. Alerts feed (AlertCard stack)
   - Section label: "ALERTS" — 11px, --text3, uppercase
   - Max 3 AlertCards, prioritize actionable (shift requests, time-off requests) over informational (cert warnings)
   - Each AlertCard tappable → navigates to team tab for request alerts, hours tab for budget alerts
   - Alert types: warning (cert expiry = amber dot), error (budget exceeded = red dot), info (new request = blue dot)

**Foreman Team Tab — Layout (top to bottom):**

1. Crew list section
   - Section label: "CREW" + crew count — 11px, --text3, uppercase
   - Each crew member: PremiumCard info variant — name (bold), role (--text2), clock status badge

2. Pending Requests section (FSCH-02, FSCH-03)
   - Section label: "PENDING REQUESTS" + count — 11px, --text3, uppercase
   - Mixed list of shift pickup requests and time-off requests (unified, sorted newest first)
   - Each request card: PremiumCard info variant
     - Employee name: --text-base, --weight-bold
     - Request type label: "SHIFT REQUEST" or "TIME OFF" — 11px uppercase, --accent for shift, --yellow for time-off
     - Date/time range: --text-base, --text2
     - Approve button: FieldButton primary, 44px height, "Approve"
     - Deny button: FieldButton danger, 44px height, "Deny"
   - Tapping Approve → opens approval bottom sheet (optional comment field + confirm button)
   - Tapping Deny → opens denial bottom sheet (optional comment field + confirm button)
   - Empty state: "No pending requests" + "Your crew is all set" body, no action button needed

3. Crew Certifications section (CRED-04)
   - Section label: "CREW CERTIFICATIONS" — 11px, --text3, uppercase
   - Filter chips: "All", "Expiring", "Expired" — horizontal scroll, flex gap var(--space-2)
     - Active chip: background rgba(var(--accent), 0.15), border 1px solid var(--accent), text var(--accent)
     - Inactive chip: background var(--bg3), border 1px solid var(--border), text var(--text2)
   - Each crew member: CredentialCard with cert status summary (active count, expiring count, expired count)
   - View-only: no edit controls on any cert card
   - Empty state when filter returns no results: "No [expiring/expired] credentials" + "Your crew certs are up to date"

## Interaction Contracts

### Bottom Sheet — Approval Comments (Approve / Deny actions)

Reuses PortalTabBar sheet overlay pattern. Triggered by Approve or Deny button tap.

- Overlay: semi-transparent backdrop, tap to cancel
- Sheet: slides up from bottom, max-height 50vh, background var(--bg3)
- Content: request summary (read-only), optional comments FieldInput (inputmode="text", placeholder "Add comment (optional)"), confirm FieldButton (primary for Approve, danger for Deny)
- Confirm button: 44px height, full width
- Cancel: "Cancel" text button at top right, --text2 color, 44px tap target
- Transition: var(--transition-state) (300ms) transform slide-up

### Navigation — every stat taps somewhere

| Tappable element | Navigates to |
|------------------|-------------|
| Driver route count hero | route tab |
| Driver Pending stat tile | route tab |
| Driver Completed stat tile | completed tab |
| Driver alert card | route tab |
| Foreman clock hero | clock tab (in More overflow) |
| Foreman Budget KPI | hours tab |
| Foreman Hours KPI | hours tab |
| Foreman Crew KPI | team tab |
| Foreman alert card (request) | team tab |
| Foreman crew member card | no-op (view only) |
| Foreman cert card | no-op (view-only per D-19) |

### Filter Chips — Crew Certifications

Single-select. Tapping an active chip does nothing (no deselect). Tapping inactive chip switches filter. No animation required beyond instant re-render. Chip row scrolls horizontally if it overflows viewport width (overflow-x: auto, no scrollbar visible).

### Header Replacement — ForemanView

Replace the inline `<header className="employee-header">` (line 750) and the inline project selector `<select className="foreman-project-select">` (line 765) with:
- `<PortalHeader variant="foreman" ...>` with projectSelector prop wired to the existing select element
- Network banner renders automatically from PortalHeader when network.online is false

### FieldSignaturePad — Inline Replacement

ForemanView lines 18–64 contain an inline FieldSignaturePad with hardcoded strokeStyle `#1e2d3b`. Replace with import from `src/components/field/FieldSignaturePad.jsx` which reads theme color via getComputedStyle. This is a cleanup task, not a user-visible feature.

## Empty + Error States

Every section with async data must render one of these states. Use EmptyState component with actionable button per VIS-05.

| Section | Empty Heading | Empty Body | Action |
|---------|--------------|------------|--------|
| Driver Home — no stops | No deliveries scheduled | Check back when your route is assigned | none (display-only when no route assigned) |
| Driver Route — no stops | Route is clear | All deliveries complete for today | none |
| Driver Completed — nothing today | No deliveries yet | Your completed deliveries will appear here | none |
| Foreman Pending Requests — empty | No pending requests | Your crew is all set | none |
| Foreman Crew Certs — filter empty (expiring) | No expiring credentials | Your crew certs are up to date | none |
| Foreman Crew Certs — filter empty (expired) | No expired credentials | All crew credentials are current | none |

Error state pattern (applies to all async sections): "Could not load [section name]" heading + "Pull down to retry" body. No action button — user retries via pull-to-refresh or returning to tab.
