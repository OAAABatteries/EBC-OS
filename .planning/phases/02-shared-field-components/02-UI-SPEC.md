# Phase 2 — UI Design Contract: Shared Field Components

> Visual and interaction contract for Phase 2. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** All 11 components in `src/components/field/`. This contract governs the visual and interaction behavior of each component in isolation. Portal wiring happens in Phases 3–5 and is out of scope here.

## Spacing Scale

Declared values — inherited from Phase 1 `src/tokens.css`. All components in this phase MUST use these tokens exclusively. No pixel literals in component source.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `--space-1` | 4px | Icon gap inside buttons, badge internal padding |
| `--space-2` | 8px | Tight intra-component spacing (label-to-input gap, tab icon-to-label gap) |
| `--space-3` | 12px | PortalHeader internal padding, MaterialRequestCard detail rows |
| `--space-4` | 16px | Default component padding (FieldCard, FieldInput) |
| `--space-5` | 20px | PortalHeader height padding, modal-equivalent sections |
| `--space-6` | 24px | FieldCard content area padding, AsyncState empty state vertical rhythm |
| `--space-8` | 32px | PortalTabBar internal spacing, section breaks |
| `--space-10` | 40px | Large vertical gaps between component groups |
| `--space-12` | 48px | Bottom safe-area offset, PortalTabBar height clearance |

Exceptions:
- **Touch targets**: `--touch-min: 44px` — enforced via `.touch-target` utility class on ALL interactive elements in this phase. This is not a spacing token; it is an interaction safety floor.
- **PortalTabBar height**: 56px fixed — accommodates 44px touch target + 12px vertical padding. Use `height: 56px` with `padding: var(--space-2) 0`.
- **PortalTabBar with safe-area**: `padding-bottom: calc(var(--space-2) + env(safe-area-inset-bottom))` for iOS notch clearance.

## Typography

Inherited from Phase 1 `src/tokens.css`. No new type tokens introduced in this phase.

| Role | Token | Value | Weight Token | Line Height | Usage in this phase |
|------|-------|-------|--------------|-------------|---------------------|
| Label / Badge | `--text-sm` | 11px | `--weight-bold` (700) | `--leading-tight` (1.2) | StatusBadge, tab labels, form labels, EmptyState sub-copy |
| Body | `--text-base` | 13px | `--weight-normal` (400) | `--leading-normal` (1.5) | FieldInput value text, MaterialRequestCard body, AsyncState messages |
| Heading / Modal title | `--text-lg` | 18px | `--weight-bold` (700) | `--leading-tight` (1.2) | PortalHeader title, EmptyState heading, FieldCard section titles |
| Display / KPI | `--text-display` | 28px | `--weight-bold` (700) | `--leading-tight` (1.2) | MaterialRequestCard quantities only — reserved for single prominent figures |

Rules:
- Exactly 2 weights in use across all components: `--weight-normal` (400) and `--weight-bold` (700). No 500, 600, or other intermediate weights.
- No `font-size` pixel literals anywhere in component source. Use `var(--text-*)` only.
- Tab bar labels: `--text-sm` at `--weight-bold`. Never icon-only (CONTEXT.md D-07).
- Form labels: `--text-sm` at `--weight-bold`, `text-transform: uppercase`, `letter-spacing: var(--tracking-wide)` — matches existing `.form-label` pattern.

## Color

This phase is **100% theme-aware**. Zero hard-coded hex, rgba, or color literals in any component source. All colors resolve from CSS custom properties that the theme system applies at runtime via `THEMES[theme].vars` on `document.documentElement.style`.

| Role | Token | Usage |
|------|-------|-------|
| Dominant surface (60%) | `var(--bg)` | Component backgrounds where full-page context needed |
| Secondary surface (30%) | `var(--bg2)` / `var(--glass-bg)` | FieldCard, PortalHeader, MaterialRequestCard, PortalTabBar backgrounds |
| Tertiary surface | `var(--bg3)` | FieldInput background, skeleton shimmer base, AsyncState empty area |
| Accent (10%) | `var(--amber)` | See accent reserved list below |
| Border | `var(--border)` / `var(--glass-border)` | All component borders — use `--border` for solid, `--glass-border` for glass surfaces |
| Body text | `var(--text)` | Primary readable content |
| Secondary text | `var(--text2)` | Supporting copy, inactive tab labels |
| Muted text | `var(--text3)` | Form labels, placeholder text, empty-state sub-copy |
| Destructive | `var(--red)` / `var(--red-dim)` | FieldButton danger variant only |
| Status: approved | `var(--status-approved)` → `var(--green)` | StatusBadge approved state |
| Status: pending | `var(--status-pending)` → `var(--amber)` | StatusBadge pending state |
| Status: denied | `var(--status-denied)` → `var(--red)` | StatusBadge denied state |
| Status: in-transit | `var(--status-in-transit)` → `var(--blue)` | StatusBadge in-transit state |
| Focus ring | `var(--focus-ring)` | All interactive elements via `.focus-visible` utility class |

**Accent reserved for (exhaustive list):**
- Active tab indicator in `PortalTabBar` (active tab icon + label color)
- `FieldButton` primary variant background
- `PortalHeader` logo text
- `LoadingSpinner` stroke / animation color
- `FieldSignaturePad` active stroke color (replaces hard-coded `#1e2d3b`)
- Badge count on `PortalTabBar` items (uses `var(--red)` for counts, `var(--amber)` for active state only)

**FieldSignaturePad specific:** Replace `ctx.strokeStyle = "#1e2d3b"` with `var(--text)` resolved at draw time (read from `getComputedStyle`). Replace `background: "#f8f9fb"` with `var(--bg2)` resolved at draw time. This is the primary extraction task for this component (CONTEXT.md D-03, canonical_refs).

## Component Interaction Contracts

Each component's visual states and interaction rules.

### FieldButton (COMP-01)

| State | Visual |
|-------|--------|
| Default (primary) | `bg: var(--amber)`, `color: #000`, `box-shadow: 0 2px 8px var(--amber-glow)` — extends `.btn .btn-primary` |
| Default (ghost) | `bg: transparent`, `border: 1px solid var(--border)`, `color: var(--text2)` — extends `.btn .btn-ghost` |
| Default (danger) | `bg: var(--red-dim)`, `color: var(--red)`, `border: 1px solid transparent` — extends `.btn .btn-danger` |
| Hover | Existing `.btn-*:hover` rules apply — no new rules needed |
| Active / pressed | `transform: scale(0.97)` — inherits from `.btn:active` |
| Loading | Spinner replaces button label; button remains full-width, disabled, `opacity: 0.7` |
| Disabled | `opacity: 0.45`, `cursor: not-allowed`, `pointer-events: none` |
| Focus | `box-shadow: var(--focus-ring)` via `.focus-visible` |

Touch target: `min-height: var(--touch-min)` — overrides `.btn` default `min-height: 36px`. Add `.touch-target` class or inline `min-height: var(--touch-min)` to the base `.btn` extension.

Loading spinner: Lucide `Loader2` icon, `animation: spin 1s linear infinite`, `--text-base` size (13px), centered within button.

### FieldCard (COMP-02)

Extends `.card`. Adds:
- Padding: `var(--space-4)` (16px) — overrides `.card` default `20px` for tighter mobile layout
- No hover transform on mobile (touch devices do not need hover lift). Use `@media (hover: hover)` to gate the `:hover` transform.
- Variants: `default` (`.card`) and `glass` (`.card-glass`) — pass as `variant` prop

### FieldInput / FieldSelect (COMP-03)

| State | Visual |
|-------|--------|
| Default | Extends `.form-input` / `.form-select` |
| Focus | `border-color: var(--amber)`, `box-shadow: 0 0 0 2px var(--amber-dim)` — inherits from `.form-input:focus` |
| Error | `border-color: var(--red)`, `box-shadow: 0 0 0 2px var(--red-dim)` |
| Disabled | `opacity: 0.45`, `cursor: not-allowed` |

Touch target: `min-height: var(--touch-min)` on the input element. This is above the current `.form-input` `padding: 8px 12px` default — adjust padding to achieve 44px total height.

`inputmode` is required on every numeric field (COMP-03, EMPL-04). The component must accept an `inputMode` prop and pass it through to the underlying `<input>`.

Error message: `--text-sm` (11px) at `var(--red)`, positioned below the input with `margin-top: var(--space-1)`.

### PortalHeader (COMP-04)

Height: 54px — matches existing `.header` class. Extends `.header` (glass background, sticky, z-index: 100).

Three variant configurations (CONTEXT.md D-01 — Claude's Discretion on pattern, but visual contract is fixed):
- **Foreman variant**: Logo left + phase/project title center + settings gear right
- **Employee variant**: Logo left + user name right + language toggle + logout
- **Driver variant**: Logo left + user name right + logout

Logo text: `var(--font-head)`, `--text-lg` (18px), `var(--amber)`, `--weight-bold` (700) — matches existing `.logo` class.

Project selector (CONTEXT.md D-02 — Claude's Discretion on placement): If placed inside the header, it must not cause the header to exceed 54px height. Recommended: place below the header as a sticky sub-header strip at 36px height using `var(--bg3)` background. This avoids 375px overflow.

### PortalTabBar (COMP-05)

Bottom-anchored. Always visible. Consumer app convention (CONTEXT.md D-07, D-08).

| Property | Value |
|----------|-------|
| Height | 56px + `env(safe-area-inset-bottom)` |
| Background | `var(--glass-bg)` with `backdrop-filter: blur(24px) saturate(1.8)` |
| Border | `border-top: 1px solid var(--glass-border)` |
| Position | `position: fixed; bottom: 0; left: 0; right: 0; z-index: 100` |
| Max primary tabs | 5 (4 recommended for most portals, never exceed 5) |
| Overflow pattern | "More" tab (5th position) — opens bottom sheet / drawer with remaining items |

Tab item states:
| State | Visual |
|-------|--------|
| Default | Icon `var(--text3)`, label `var(--text3)`, `--text-sm` (11px), `--weight-normal` |
| Active | Icon `var(--amber)`, label `var(--amber)`, `--weight-bold` (700) |
| Badge | Red dot `var(--red)` at top-right of icon, 6px diameter |
| Pressed | `transform: scale(0.92)`, `transition: var(--transition-micro)` |
| Focus | `box-shadow: var(--focus-ring)` via `.focus-visible` |

Icon + label requirement: Icons are 20px Lucide SVG. Label is `--text-sm` (11px). Tab item layout: flex column, icon top, label bottom, gap `var(--space-1)` (4px). Touch target: min 44px × 44px per tab item.

"More" overflow: Opens a bottom sheet (slide-up panel from bottom edge). Sheet uses `var(--glass-bg)` background, lists overflow tabs as full-width rows with icon + label + optional badge. Sheet closes on backdrop tap or second "More" tap.

### StatusBadge (COMP-06)

Extends `.badge`. Semantic mapping:

| Status value | CSS class | Rendered token |
|---|---|---|
| `approved` | `.badge-green` | `var(--status-approved)` → `var(--green)` |
| `pending` | `.badge-amber` | `var(--status-pending)` → `var(--amber)` |
| `denied` | `.badge-red` | `var(--status-denied)` → `var(--red)` |
| `in-transit` | `.badge-blue` | `var(--status-in-transit)` → `var(--blue)` |
| `completed` | `.badge-muted` | `var(--text3)` |
| `project` | `.badge-muted` | `var(--status-project)` → `var(--text2)` |

No new CSS classes needed — `StatusBadge` is a thin React wrapper that maps a `status` string prop to an existing `.badge-*` class. Do not invent new badge colors.

### EmptyState (COMP-07)

Layout: Centered column, `var(--space-6)` (24px) vertical padding, `var(--space-4)` (16px) horizontal padding.

| Element | Style |
|---------|-------|
| Icon | Lucide, 48px, `var(--text3)` color |
| Heading | `--text-lg` (18px), `--weight-bold` (700), `var(--text)` |
| Body copy | `--text-base` (13px), `--weight-normal` (400), `var(--text2)`, `--leading-normal` (1.5) |
| Action button | `FieldButton` ghost variant — optional, not always present |

### LoadingSpinner / Skeleton (COMP-08)

**Spinner:** Lucide `Loader2`, 20px default, `var(--amber)` color, `animation: spin 1s linear infinite`. Used inline inside `FieldButton` when loading, and as standalone centered spinner for full-section loads.

**Skeleton:** Block-level shimmer placeholder. Background: `var(--bg3)`. Shimmer animation: `background: linear-gradient(90deg, var(--bg3) 0%, var(--bg2) 50%, var(--bg3) 100%)` animated left-to-right over 1.5s. Border-radius: `var(--radius-sm)` (inherited from theme). Dimensions: match the shape of the content they replace (skeleton lines for text, skeleton blocks for cards).

No hard-coded colors in skeleton animation. Use CSS variable references only.

### AsyncState (COMP-09)

Four mutually exclusive render states. Children wrapper pattern (CONTEXT.md D-05 — Claude's Discretion confirmed as children wrapper for compatibility with existing portal code):

| State | Renders |
|-------|---------|
| `loading` | `LoadingSpinner` (centered) or skeleton (if `skeleton` prop provided) |
| `empty` | `EmptyState` with `message` prop and optional `action` prop |
| `error` | Error display (see copywriting contract below) |
| `success` | `children` |

Props: `loading: bool`, `empty: bool`, `error: string | null`, `emptyMessage: string`, `emptyAction: ReactNode`, `skeleton: ReactNode`. If `loading` is true, all other states are ignored. If `error` is set, error state takes priority over `empty`.

### FieldSignaturePad (COMP-10)

Extracted from `ForemanView.jsx` lines 20–65. Logic is preserved exactly — only color constants are tokenized.

| Hard-coded value to replace | Replacement |
|-----------------------------|-------------|
| `ctx.strokeStyle = "#1e2d3b"` | `ctx.strokeStyle = getComputedStyle(canvasRef.current).getPropertyValue('--text').trim()` |
| `background: "#f8f9fb"` | `background: var(--bg2)` (CSS, not canvas context) |

Canvas border: `1px solid var(--border)`, `border-radius: var(--radius)`.
Action buttons (Clear, Save): `FieldButton` ghost and primary variants respectively. Minimum 44px touch targets.
Canvas minimum height: 160px on mobile (375px viewport), 200px on wider.

### MaterialRequestCard (COMP-11)

Extends `FieldCard`. Renders identically across Foreman, Employee, and Driver portals.

Layout: Card with header row (request title + `StatusBadge`), body rows (material name, quantity, unit), footer row (submitted by + timestamp + optional action buttons).

| Element | Style |
|---------|-------|
| Title | `--text-lg` (18px), `--weight-bold` (700), `var(--text)` |
| StatusBadge | `StatusBadge` component, right-aligned in header row |
| Material name | `--text-base` (13px), `--weight-bold` (700), `var(--text)` |
| Quantity | `--text-display` (28px), `--weight-bold` (700), `var(--amber)` — prominent field figure |
| Unit label | `--text-sm` (11px), `--weight-normal` (400), `var(--text3)` |
| Submitted by | `--text-sm` (11px), `--weight-normal` (400), `var(--text2)` |
| Timestamp | `--text-sm` (11px), `--weight-normal` (400), `var(--text3)` |
| Action buttons | `FieldButton` (approve=primary, deny=danger, ghost=secondary actions) |

## Copywriting Contract

All display text must pass through the `t()` translation function. Components accept `t` as a prop. No hardcoded English or Spanish strings in component JSX.

| Element | English copy | Notes |
|---------|-------------|-------|
| EmptyState — no data default heading | "Nothing here yet" | Generic fallback; callers override via `emptyMessage` prop |
| EmptyState — no data default body | "Check back later or contact your foreman." | Generic fallback |
| EmptyState — no assignments | "No assignments right now." | Used in DriverView/EmployeeView context (passed by caller) |
| AsyncState error heading | "Something went wrong." | Short, non-technical |
| AsyncState error body | "Pull down to refresh, or contact support if this keeps happening." | Actionable next step |
| FieldSignaturePad — clear button | "Clear" | Verb, no noun needed — action is obvious in context |
| FieldSignaturePad — save button | "Save Signature" | Verb + noun |
| PortalTabBar "More" tab label | "More" | Matches Facebook/TikTok convention (CONTEXT.md D-06) |
| FieldInput error message | "[Field name] is required." | Caller provides; component just renders `errorMessage` prop |
| MaterialRequestCard — no action state | No copy needed — StatusBadge communicates state | — |
| LoadingSpinner aria-label | "Loading" | Screen reader only |
| FieldButton loading state aria-label | "Loading, please wait" | Screen reader only |

Destructive actions in this phase:
- **FieldSignaturePad "Clear"**: Clears canvas in-place. No confirmation dialog — the canvas is a transient capture, not persisted data. User sees the cleared canvas immediately and can redraw.
- **FieldButton danger variant**: No built-in confirmation — confirmation is the caller's responsibility. The component does not own the destructive decision.

## Implementation Notes for Executor

1. **CSS location**: Add all new component utility classes to `src/styles.js` — not in separate `.css` files per component. Follows established codebase pattern (CONTEXT.md code_context).
2. **Component directory**: `src/components/field/` — does not exist yet. Create it. No subdirectories needed.
3. **Index file**: Create `src/components/field/index.js` with named exports for all 11 components.
4. **Translation prop**: Every component that renders visible text accepts `t` as a required prop. Do not use a hook — prop threading is the existing portal pattern.
5. **Theme participation**: No component-level theme logic. Components use `var(--token)` CSS references. Theme switching happens in `App.jsx` and the cascade resolves automatically.
6. **No new npm packages**: Lucide React is already installed. No other new dependencies are permitted (CONTEXT.md hard constraint).
7. **Existing class extension rule**: When a component extends an existing utility class (`.btn`, `.card`, `.badge`), apply the base class via `className` composition — do not copy the CSS rules into a new class.
