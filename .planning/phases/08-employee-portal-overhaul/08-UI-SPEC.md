## Spacing Scale

Inherited from Phase 7 (tokens.css). No new tokens added this phase. All values are px — no rem.

| Token | Value | Usage in Phase 8 |
|-------|-------|-----------------|
| --space-1 | 4px | Icon gaps, dot indicators, week strip day-dot margin |
| --space-2 | 8px | Badge internal padding, stat tile row gap, credential badge gap |
| --space-3 | 12px | Info Card and Alert Card internal padding, bottom sheet item gap |
| --space-4 | 16px | Hero Card padding, Home page section padding, bottom sheet header padding |
| --space-5 | 20px | Between-card vertical gaps on Home and Schedule tabs |
| --space-6 | 24px | Section header-to-content gap (e.g. "ALERTS" label to first AlertCard) |
| --space-8 | 32px | Between major sections (hero to stat row, stat row to project card) |
| --space-10 | 40px | Page-level top padding below PortalHeader |
| --space-12 | 48px | Reserved, bottom-of-page clearance above PortalTabBar |

Exceptions:
- Touch targets: 44px minimum via --touch-min (pre-existing token). All interactive elements — stat tiles, alert cards, week strip day buttons, shift "Pick Up" buttons, time-off trigger, sheet close button.
- Week strip day cell: 40px × 40px tap target (40px meets near-min; compensate with --space-2 gap between cells so thumb has visual clearance). Do not shrink below 40px.
- Bottom sheet overlay: full viewport height. Sheet slides to 50vh–85vh depending on form content.
- PortalHeader height: 48px (locked, pre-existing).
- "Last updated" timestamp label: 11px (--text-sm), sits flush below week strip at --space-2 below the strip.

Project-override declaration: This project extends the standard 8-point set {4, 8, 16, 24, 32, 48, 64} with four additional values — 12px (--space-3), 20px (--space-5), 40px (--space-10), and 44px (--touch-min). All four are multiples of 4. They were established in Phase 1 tokens.css and carried forward unchanged through Phase 7. They are not new deviations introduced in Phase 8 — they are inherited project tokens. This declaration is the explicit record of that project decision, consistent with the precedent set in 07-UI-SPEC.md Spacing Scale ("Non-canonical values note").

Source: tokens.css (direct read), 07-UI-SPEC.md Spacing Scale section

## Focal Point

Primary screen focal point on Home tab: clock status hero card — "ON CLOCK" / "OFF CLOCK" at --text-display (28px, --weight-bold) in a PremiumCard Hero variant with --accent left border. This is the first element the employee reads on open. Everything else (stat row, project card, alerts) is secondary and flows below.

Tab-level focal points:
- Schedule: the week strip row — today's date numeral in --accent, current day cell highlighted
- Credentials: the sort-to-top expired/expiring cards — red/amber badges command first attention
- Drawings: inherited DrawingsTab behavior (file list, no special focal hierarchy added this phase)

Source: CONTEXT.md D-05, D-06, D-15; 07-UI-SPEC.md Focal Point section

## Tab Layout Contract

Primary bottom bar tabs (maxPrimary=5): Home, Clock, Schedule, Materials, More.

More overflow panel contains (in order): Credentials, Drawings, Time Log, JSA, Change Orders, RFIs, Settings.

Default tab on mount: "home" (empTab useState default changed from "clock" to "home").

Settings back-button target: "home" (changed from "clock").

Credentials badge: shown when any cert is expired OR expiring within 30 days. Use --red for expired, --yellow for expiring. Badge count = number of affected certs.

Materials badge: inherited behavior (badge when any request has status "requested").

Clock badge: inherited behavior (badge = isClockedIn indicator).

## Interaction Contract

| Interaction | Behavior | Timing |
|------------|----------|--------|
| App open / mount | Land on Home tab — clock hero, stat row, project card, alerts feed render | instant |
| Stat tile tap (Hours) | setEmpTab("log") — switch to Time Log tab | instant |
| Stat tile tap (Tasks) | setEmpTab("schedule") — switch to Schedule tab | instant |
| Stat tile tap (Pending) | setEmpTab("materials") — switch to Materials tab | instant |
| Alert card tap | setEmpTab(alert.sourceTab) — routes to credentials/materials/schedule | instant |
| Clock hero card tap | setEmpTab("clock") — switch to Clock tab | instant |
| "View All" alerts link tap | Expand alerts feed to show all items inline (no navigation, no new sheet) | instant |
| Return to Home from Clock | useEffect detects empTab==="home" after prevTab==="clock" — stats re-derive automatically from memos, no explicit refetch needed unless Supabase data is stale | instant |
| Week strip day tap | setSelectedDay(day) — updates shifts list below strip (no scroll animation, instant re-render) | instant |
| Week strip swipe left | Navigate to next week (increment weekOffset by 7) | --duration-state (300ms) slide |
| Week strip swipe right | Navigate to previous week (decrement weekOffset by 7) | --duration-state (300ms) slide |
| Shift "Pick Up" button tap | Disable button + show FieldButton loading spinner, insert to shift_requests with status="pending", update ShiftCard status inline to "PENDING", re-enable or show success state | --duration-micro (150ms) for press visual |
| "Request Time Off" button tap | Open time-off bottom sheet — sheet slides up | --duration-state (300ms) slide up |
| Bottom sheet overlay tap | Close sheet — slides down | --duration-state (300ms) slide down |
| Bottom sheet Submit tap | FieldButton loading spinner, insert to time_off_requests, dismiss on success, show inline error copy on failure (do not close sheet on fail) | --duration-micro (150ms) for press |
| "Add Credential" button tap | Open add-credential bottom sheet — same sheet pattern as time-off | --duration-state (300ms) slide up |
| Credential card tap | Expand credential detail inline (accordion) or navigate to a detail view — Claude's Discretion: use accordion expand within CredentialCard (no new tab/page) | --duration-state (300ms) height transition |
| Schedule offline state | Show "Last updated HH:MM" in .schedule-offline-notice below week strip. Data renders from localStorage cache. No spinner — data is present. | instant |
| Drawings tab render | Mount DrawingsTab with readOnly={true} and projectFilter={assignedProjectId} — inherits DrawingsTab's own loading/error/empty states | DrawingsTab-managed |
| EmptyState action button | Always present. No instance renders without actionable button. | instant |
| All Supabase queries | Wrapped in AsyncState or manual loading/error/empty state. Error state shows error copy + retry button. | n/a |
| Bottom sheet focus trap | Focus trapped within sheet when open. Escape key closes sheet. Background scroll disabled while sheet is open (overflow:hidden on body). | n/a |
| Touch target minimum | 44px via --touch-min on all interactive elements. Week strip day cells: 40px (documented exception). | n/a |

PortalTabBar maxPrimary: set to 5. Reasoning: 4 primary + More creates a cramped 4-tab bar with too much white space and requires an extra tap for Clock. 5 primary fits the 4 daily-use tabs (Home, Clock, Schedule, Materials) plus More without crowding at 375px (each tab = ~75px).

Available shifts placement: rendered as a separate section below the day's scheduled shifts, separated by a .section-label divider "AVAILABLE SHIFTS". This avoids a sub-tab toggle that adds navigation complexity. If no available shifts, the section is hidden entirely (not an EmptyState).

Stat tile navigation: uses tab switch (setEmpTab) not route change. No React Router in this project — tab state is prop-drilled via setEmpTab.

Source: CONTEXT.md D-01 through D-18, RESEARCH.md Patterns 1-5, 07-UI-SPEC.md Interaction Contract

## Accessibility Notes

All inherited from Phase 7 plus Phase 8 specifics:

- All interactive elements: 44px minimum via --touch-min. Week strip day cells: 40px (documented exception, acceptable given 375px viewport constraint).
- Focus rings on all interactive elements via --focus-ring token (Phase 1, enforced).
- Color is never the only status differentiator — StatusBadge uses text label + color, credential status uses text badge + color, week strip uses text + color.
- Week strip today cell: aria-label="Today, {day abbreviation} {date}" on the day cell button.
- Bottom sheets: focus trap within sheet when open. Escape closes sheet. body overflow:hidden while sheet is open.
- Alert cards: aria-label="{alert type}: {message}" to give full context to screen readers.
- Lucide icons: aria-hidden="true" where decorative. Meaningful standalone icons (e.g. close button "X") get aria-label="Close".
- Stat tiles: role="button" or button element with aria-label="{label}: {value}, tap to view details".
- Empty states: action button must have descriptive label (not just "Go" or "OK").
- "View All" link: minimum 44px tap target even if visually small — use padding to meet target.
- OVERTIME badge: supplemented by text, not color alone — badge text "OVERTIME" present on all overtime shifts.

Source: 07-UI-SPEC.md Accessibility Notes; CONTEXT.md (touch/interaction constraints)
